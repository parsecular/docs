## Error Response Format

All error responses return a JSON object with three fields:

```json
{
  "error": "human-readable message",
  "code": "machine_readable_code",
  "retryable": false
}
```

| Field | Type | Description |
|-------|------|-------------|
| `error` | string | Human-readable error message |
| `code` | string | Machine-readable error code (see table below) |
| `retryable` | boolean | `true` if the request can be safely retried (only for `rate_limit_exceeded` and `service_unavailable`) |

### Error Codes

| Code | Meaning |
|------|---------|
| `unauthorized` | Missing or invalid API key |
| `credentials_required` | Exchange credentials not configured |
| `authentication_failed` | Exchange-level auth failure |
| `missing_parameter` | Required parameter missing |
| `bad_request` | Invalid request format or parameters |
| `exchange_not_found` | Unknown exchange identifier |
| `market_not_found` | Market does not exist |
| `order_not_found` | Order does not exist |
| `not_found` | Generic resource not found |
| `insufficient_funds` | Not enough balance for the operation |
| `not_supported` | Exchange does not support this operation |
| `rate_limit_exceeded` | Too many requests (retryable) |
| `service_unavailable` | Upstream exchange down (retryable) |
| `internal_error` | Unexpected server failure |

## HTTP Status Codes

| Status | Meaning | Action |
|--------|---------|--------|
| `400` | Bad request — invalid parameters or missing required fields | Fix the request and retry |
| `401` | Unauthorized — missing or invalid API key | Check your `X-API-Key` header |
| `402` | Insufficient funds — not enough balance for the operation | Deposit funds on the exchange |
| `403` | Forbidden — credentials or permissions issue for the target exchange | Verify your exchange credentials in Settings |
| `404` | Not found — the requested resource doesn't exist | Check the ID/ticker you're using |
| `429` | Rate limited — too many requests | Wait and retry (SDKs handle this automatically) |
| `500` | Internal server error — unexpected failure | Retry; contact support if persistent |
| `501` | Not supported — the exchange doesn't support this operation | Check exchange capabilities via `/exchanges` |
| `503` | Service unavailable — upstream exchange is down or unreachable | Retry with backoff |

## Rate Limiting

When rate limited, the response includes a `Retry-After` header plus your current tier and QPS limit:

```
HTTP/1.1 429 Too Many Requests
Retry-After: 1
Content-Type: application/json

{ "error": "rate limit exceeded", "code": "rate_limit_exceeded", "retryable": true, "tier": "free", "limit_qps": 10 }
```

Both SDKs automatically retry on `429` responses with exponential backoff (up to 2 retries).

## SDK Error Handling

<CodeGroup>
```typescript TypeScript
import ParsecAPI from 'parsec-api';

const client = new ParsecAPI();

try {
  const order = await client.orders.create({
    exchange: 'polymarket',
    market_id: '0x...',
    outcome: 'yes',
    side: 'buy',
    price: 0.55,
    size: 100,
  });
} catch (err) {
  if (err instanceof ParsecAPI.APIError) {
    console.log(err.status);  // 400, 401, 403, 429, etc.
    console.log(err.message); // Error message from server
  }
}
```
```python Python
from parsec_api import ParsecAPI, APIStatusError

client = ParsecAPI()

try:
    order = client.orders.create(
        exchange="polymarket",
        market_id="0x...",
        outcome="yes",
        side="buy",
        price=0.55,
        size=100,
    )
except APIStatusError as err:
    print(err.status_code)  # 400, 401, 403, 429, etc.
    print(err.message)      # Error message from server
```
</CodeGroup>

## Common Errors

### `401` — Invalid API Key

```json
{ "error": "invalid API key", "code": "unauthorized", "retryable": false }
```

```json
{ "error": "missing X-API-Key header", "code": "unauthorized", "retryable": false }
```

Make sure you're passing your API key in the `X-API-Key` header, or set the `PARSEC_API_KEY` environment variable for the SDKs.

### `403` — Exchange Credentials Issue

```json
{ "error": "exchange authentication failed: kalshi", "code": "authentication_failed", "retryable": false }
```

Your exchange credentials are missing or invalid. Go to **Settings** in your Parsec dashboard and re-enter your credentials for the affected exchange.

### `400` — Historical Data Tier Limit

```json
{ "error": "historical data limited to 5 days on the free tier — upgrade for more", "code": "bad_request", "retryable": false }
```

Your `start_ts` requests data older than your tier allows. Free: 5 days, Pro: 30 days, Scale: unlimited.

### `400` — Missing Required Parameters

```json
{ "error": "exchanges parameter is required", "code": "missing_parameter", "retryable": false }
```

Check the [API Reference](/api/list-exchanges) for required parameters on each endpoint.
