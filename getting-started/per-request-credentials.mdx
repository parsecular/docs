## Overview

With per-request credentials, nothing is stored on Parsec's servers. You pass
your exchange credentials with each request via the `X-Exchange-Credentials`
header or the body `credentials` field. Parsec creates a transient exchange
session, executes the operation, and discards the credentials immediately.

**Best for:** Security-conscious traders who don't want credentials stored
anywhere, and builders whose users manage their own keys.

```
Your App ───► POST /orders + X-Exchange-Credentials header ───► Parsec authenticates (transient)
                                                            ───► Credentials discarded after request
```

<Info>
No onboarding required. You can start making API calls immediately with just
your Parsec API key and exchange credentials.
</Info>

## How it works

Two ways to pass credentials:

### Option 1: Header (all endpoints)

Base64-encode a JSON object and pass it as `X-Exchange-Credentials`:

<CodeGroup>
```bash Kalshi
# 1. Base64-encode your credentials
CREDS=$(echo -n '{"api_key_id":"your-key-id","private_key":"-----BEGIN RSA PRIVATE KEY-----\nMIIE..."}' | base64)

# 2. Pass as header
curl -H "X-API-Key: pk_live_YOUR_KEY" \
  -H "X-Exchange-Credentials: $CREDS" \
  "https://api.parsecapi.com/api/v1/positions?exchange=kalshi"
```
```bash Polymarket
CREDS=$(echo -n '{"clob_api_key":"...","clob_api_secret":"...","clob_api_passphrase":"..."}' | base64)

curl -H "X-API-Key: pk_live_YOUR_KEY" \
  -H "X-Exchange-Credentials: $CREDS" \
  "https://api.parsecapi.com/api/v1/balance?exchange=polymarket"
```
</CodeGroup>

Works on all 6 private endpoints: orders (create, list, get, cancel), positions, and balance.

### Option 2: Body field (POST /orders only)

Include a `credentials` object in the request body:

```bash
curl -X POST "https://api.parsecapi.com/api/v1/orders?exchange=kalshi" \
  -H "X-API-Key: pk_live_YOUR_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "market_id": "KXBTC-26FEB18",
    "outcome": "Yes",
    "side": "buy",
    "price": 0.65,
    "size": 10,
    "credentials": {
      "api_key_id": "your-key-id",
      "private_key": "-----BEGIN RSA PRIVATE KEY-----\nMIIE..."
    }
  }'
```

<Info>
Body credentials take precedence over the header if both are present.
</Info>

## Credential shapes

### Kalshi

```json
{
  "api_key_id": "your-kalshi-key-id",
  "private_key": "-----BEGIN RSA PRIVATE KEY-----\nMIIE..."
}
```

### Polymarket

```json
{
  "clob_api_key": "your-clob-api-key",
  "clob_api_secret": "your-clob-api-secret",
  "clob_api_passphrase": "your-clob-passphrase",
  "private_key": "0xYourEthPrivateKey"
}
```

| Field | Required | Description |
|-------|----------|-------------|
| `clob_api_key` | **Yes** | CLOB API key |
| `clob_api_secret` | **Yes** | CLOB API secret |
| `clob_api_passphrase` | **Yes** | CLOB API passphrase |
| `private_key` | No | Ethereum private key (`0x`-prefixed hex) for order signing. Without it, only read operations work |

## SDK examples

<CodeGroup>
```typescript TypeScript
import ParsecAPI from 'parsec-api';

const client = new ParsecAPI({ apiKey: 'pk_live_YOUR_KEY' });

// Option A: Pass credentials per-request in the body
const order = await client.orders.create({
  exchange: 'kalshi',
  market_id: 'KXBTC-26FEB18',
  outcome: 'Yes',
  side: 'buy',
  price: 0.65,
  size: 10,
  credentials: {
    api_key_id: 'your-key-id',
    private_key: '-----BEGIN RSA PRIVATE KEY-----\nMIIE...',
  },
});

// Option B: Set header globally for all requests
const creds = btoa(JSON.stringify({
  api_key_id: 'your-key-id',
  private_key: '-----BEGIN RSA PRIVATE KEY-----\nMIIE...',
}));
const globalClient = new ParsecAPI({
  apiKey: 'pk_live_YOUR_KEY',
  defaultHeaders: { 'X-Exchange-Credentials': creds },
});
```
```python Python
from parsec_api import ParsecAPI
import base64, json

client = ParsecAPI(api_key="pk_live_YOUR_KEY")

# Option A: Pass credentials per-request in the body
order = client.orders.create(
    exchange="kalshi",
    market_id="KXBTC-26FEB18",
    outcome="Yes",
    side="buy",
    price=0.65,
    size=10,
    credentials={
        "api_key_id": "your-key-id",
        "private_key": "-----BEGIN RSA PRIVATE KEY-----\nMIIE...",
    },
)

# Option B: Set header globally for all requests
creds = base64.b64encode(json.dumps({
    "api_key_id": "your-key-id",
    "private_key": "-----BEGIN RSA PRIVATE KEY-----\nMIIE...",
}).encode()).decode()
global_client = ParsecAPI(
    api_key="pk_live_YOUR_KEY",
    default_headers={"X-Exchange-Credentials": creds},
)
```
</CodeGroup>

## Supported endpoints

| Endpoint | Method | Credential Source |
|----------|--------|-------------------|
| `/api/v1/orders` | POST | body or header |
| `/api/v1/orders` | GET | header |
| `/api/v1/orders/:id` | GET | header |
| `/api/v1/orders/:id` | DELETE | header |
| `/api/v1/positions` | GET | header |
| `/api/v1/balance` | GET | header |
