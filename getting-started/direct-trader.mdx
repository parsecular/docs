## Overview

With self credentials, you provide your own exchange API keys and Parsec stores
them securely. All subsequent API calls use these stored credentials — no
per-request auth needed beyond your Parsec API key.

**Best for:** Traders who already have exchange accounts and API keys.

```
Your App ───► POST /onboard { mode: "self", credentials... } ───► Parsec stores credentials
         ───► POST /orders                                    ───► Parsec authenticates and routes
```

## Step 1: Get your API key

Sign up at [parsecapi.com](https://parsecapi.com) → **Settings** → copy your API key (`pk_live_...`).

## Step 2: Onboard your exchange

Use `POST /api/v1/onboard` with `mode: "self"` to store your credentials.

### Kalshi

**Where to get these:**
1. Log in to [kalshi.com](https://kalshi.com) → API settings → Generate new key pair
2. Copy the **Key ID** and download the **private key file** (`.pem`)

<CodeGroup>
```bash curl
curl -X POST "https://api.parsecapi.com/api/v1/onboard" \
  -H "X-API-Key: pk_live_YOUR_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "exchange": "kalshi",
    "mode": "self",
    "api_key_id": "your-kalshi-key-id",
    "private_key": "-----BEGIN RSA PRIVATE KEY-----\nMIIE..."
  }'
```
```typescript TypeScript
import ParsecAPI from 'parsec-api';

const client = new ParsecAPI({ apiKey: 'pk_live_YOUR_KEY' });

const result = await client.onboard.create({
  exchange: 'kalshi',
  mode: 'self',
  api_key_id: 'your-kalshi-key-id',
  private_key: '-----BEGIN RSA PRIVATE KEY-----\nMIIE...',
});
```
```python Python
from parsec_api import ParsecAPI

client = ParsecAPI(api_key="pk_live_YOUR_KEY")

result = client.onboard.create(
    exchange="kalshi",
    mode="self",
    api_key_id="your-kalshi-key-id",
    private_key="-----BEGIN RSA PRIVATE KEY-----\nMIIE...",
)
```
</CodeGroup>

### Polymarket

**Where to get these:**
- Derive CLOB API credentials from your wallet at [clob.polymarket.com](https://clob.polymarket.com)

<CodeGroup>
```bash curl
curl -X POST "https://api.parsecapi.com/api/v1/onboard" \
  -H "X-API-Key: pk_live_YOUR_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "exchange": "polymarket",
    "mode": "self",
    "clob_api_key": "your-clob-api-key",
    "clob_api_secret": "your-clob-api-secret",
    "clob_api_passphrase": "your-clob-passphrase",
    "private_key": "0xYourEthPrivateKey"
  }'
```
```typescript TypeScript
const result = await client.onboard.create({
  exchange: 'polymarket',
  mode: 'self',
  clob_api_key: 'your-clob-api-key',
  clob_api_secret: 'your-clob-api-secret',
  clob_api_passphrase: 'your-clob-passphrase',
  private_key: '0xYourEthPrivateKey',
});
```
```python Python
result = client.onboard.create(
    exchange="polymarket",
    mode="self",
    clob_api_key="your-clob-api-key",
    clob_api_secret="your-clob-api-secret",
    clob_api_passphrase="your-clob-passphrase",
    private_key="0xYourEthPrivateKey",
)
```
</CodeGroup>

<Info>
The `private_key` field is optional for Polymarket self mode. Including your Ethereum
private key (`0x`-prefixed hex) enables order signing so you can place and cancel orders.
Without it, only read operations (balance, positions) work.
</Info>

### Onboard fields reference

| Field | Type | Required | Description |
|-------|------|----------|-------------|
| `exchange` | string | **Yes** | `"polymarket"` or `"kalshi"` |
| `mode` | string | **Yes** | `"self"` |
| `clob_api_key` | string | Polymarket | CLOB API key |
| `clob_api_secret` | string | Polymarket | CLOB API secret |
| `clob_api_passphrase` | string | Polymarket | CLOB API passphrase |
| `private_key` | string | Kalshi (required), Polymarket (optional) | Kalshi: RSA private key (PEM). Polymarket: Ethereum private key (`0x`-prefixed hex) for order signing |
| `api_key_id` | string | Kalshi | Kalshi API key ID |

### Response

<ResponseExample>
```json Kalshi
{
  "exchange": "kalshi",
  "mode": "self",
  "status": "complete",
  "wallet_address": null,
  "linked_exchanges": ["kalshi"],
  "steps_completed": ["credentials_validated", "credentials_stored"]
}
```
```json Polymarket
{
  "exchange": "polymarket",
  "mode": "self",
  "status": "complete",
  "wallet_address": null,
  "linked_exchanges": ["polymarket"],
  "steps_completed": ["credentials_stored"]
}
```
</ResponseExample>

| Field | Type | Always present | Description |
|-------|------|----------------|-------------|
| `exchange` | string | Yes | Exchange that was onboarded |
| `mode` | string | Yes | `"self"` |
| `status` | string | Yes | `"complete"` on success |
| `wallet_address` | string | No | Always `null` for self mode |
| `linked_exchanges` | string[] | Yes | All exchanges linked to this account |
| `steps_completed` | string[] | Yes | Steps completed during this call |

## Step 3: Verify connectivity

<CodeGroup>
```bash curl
curl -H "X-API-Key: pk_live_YOUR_KEY" \
  "https://api.parsecapi.com/api/v1/ping?exchange=kalshi"
```
```typescript TypeScript
const result = await client.ping.list({ exchange: 'kalshi' });
console.log(result); // [{ exchange: "kalshi", authenticated: true, ... }]
```
```python Python
result = client.ping.list(exchange="kalshi")
print(result)  # [PingResponse(exchange="kalshi", authenticated=True, ...)]
```
</CodeGroup>

## Step 4: Place your first order

<CodeGroup>
```bash curl
curl -X POST "https://api.parsecapi.com/api/v1/orders?exchange=kalshi" \
  -H "X-API-Key: pk_live_YOUR_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "market_id": "KXBTC-26FEB18",
    "outcome": "Yes",
    "side": "buy",
    "price": 0.65,
    "size": 10
  }'
```
```typescript TypeScript
const order = await client.orders.create({
  exchange: 'kalshi',
  market_id: 'KXBTC-26FEB18',
  outcome: 'Yes',
  side: 'buy',
  price: 0.65,
  size: 10,
});
console.log(order.id, order.status);
```
```python Python
order = client.orders.create(
    exchange="kalshi",
    market_id="KXBTC-26FEB18",
    outcome="Yes",
    side="buy",
    price=0.65,
    size=10,
)
print(order.id, order.status)
```
</CodeGroup>
