## Overview

As a builder, you manage end-users programmatically through the Builder API.
Each end-user gets their own Parsec customer ID and API key. You control
everything using your builder API key.

There are three credential strategies for your end-users:

| Strategy | When to use | Onboard? |
|----------|-------------|----------|
| **Managed** | User doesn't have a wallet — Parsec creates one | `mode: "managed"` |
| **Self** | User has exchange API keys — Parsec stores them | `mode: "self"` |
| **Per-request** | User manages their own keys — nothing stored | No onboard, pass `X-Exchange-Credentials` per-request |

## List users

`GET /api/v1/builder/users`

### Parameters

<ParamField query="limit" type="number" default="50">
  Items per page (1–100).
</ParamField>

<ParamField query="cursor" type="string">
  Pagination cursor from `next_cursor`.
</ParamField>

### Response

<ResponseField name="users" type="BuilderUserResponse[]">
  Array of user objects (same shape as the [create response](#response)).
</ResponseField>

<ResponseField name="next_cursor" type="string | null">
  Pass as `cursor` to fetch the next page. Omitted on the last page.
</ResponseField>

<ResponseField name="count" type="number">
  Number of items in this response.
</ResponseField>

<RequestExample>
```bash
curl "https://api.parsecapi.com/api/v1/builder/users?limit=20" \
  -H "X-API-Key: pk_live_YOUR_BUILDER_KEY"
```
</RequestExample>

<ResponseExample>
```json
{
  "users": [
    {
      "customer_id": "a1b2c3d4-...",
      "external_id": "user_001",
      "api_key": "pk_live_abc123...",
      "tier": "free",
      "qps_limit": 10,
      "linked_exchanges": ["polymarket"],
      "wallet": {
        "eoa_address": "0x1234...",
        "privy_wallet_id": "wallet_abc123",
        "wallet_type": "eoa"
      },
      "fee_escrow_enabled": false
    }
  ],
  "next_cursor": "eyJidWlsZGVyX2V4dGVybmFsX2lkIjoiMDAyIn0=",
  "count": 1
}
```
</ResponseExample>

## Get user

`GET /api/v1/builder/users/:customer_id`

Returns the same response shape as the [create endpoint](#response).

<RequestExample>
```bash
curl "https://api.parsecapi.com/api/v1/builder/users/CUSTOMER_ID" \
  -H "X-API-Key: pk_live_YOUR_BUILDER_KEY"
```
</RequestExample>

## Update user

`PATCH /api/v1/builder/users/:customer_id`

### Parameters

<ParamField body="qps_limit" type="number">
  New per-user rate limit (must be >= 1).
</ParamField>

Returns the updated user (same shape as the [create response](#response)).

<RequestExample>
```bash
curl -X PATCH "https://api.parsecapi.com/api/v1/builder/users/CUSTOMER_ID" \
  -H "X-API-Key: pk_live_YOUR_BUILDER_KEY" \
  -H "Content-Type: application/json" \
  -d '{ "qps_limit": 20 }'
```
</RequestExample>

## Deactivate user

`DELETE /api/v1/builder/users/:customer_id`

Deactivates all wallets, credentials, and session signers. **Not reversible** —
the user will need to be re-onboarded to trade again.

Returns `204 No Content`.

<RequestExample>
```bash
curl -X DELETE "https://api.parsecapi.com/api/v1/builder/users/CUSTOMER_ID" \
  -H "X-API-Key: pk_live_YOUR_BUILDER_KEY"
```
</RequestExample>

## QPS pool

`GET /api/v1/builder/pool`

Monitor your QPS allocation across end-users.

### Response

<ResponseField name="qps_pool" type="number">
  Total builder QPS pool (default: 500).
</ResponseField>

<ResponseField name="end_user_count" type="number">
  Number of end-users.
</ResponseField>

<ResponseField name="total_allocated_qps" type="number">
  Sum of all end-user `qps_limit` values.
</ResponseField>

<RequestExample>
```bash
curl "https://api.parsecapi.com/api/v1/builder/pool" \
  -H "X-API-Key: pk_live_YOUR_BUILDER_KEY"
```
</RequestExample>

<ResponseExample>
```json
{
  "qps_pool": 500,
  "end_user_count": 12,
  "total_allocated_qps": 120
}
```
</ResponseExample>

## Per-request credentials

For users who don't want credentials stored, skip onboarding and pass exchange
credentials per-request via the `X-Exchange-Credentials` header. See the full
[per-request credentials guide](/getting-started/per-request-credentials)
for credential shapes, supported endpoints, and SDK examples.

<RequestExample>
```bash Create user (no onboard)
curl -X POST "https://api.parsecapi.com/api/v1/builder/onboard" \
  -H "X-API-Key: pk_live_YOUR_BUILDER_KEY" \
  -H "Content-Type: application/json" \
  -d '{ "external_id": "user_005" }'
```
```bash Trade with per-request credentials
CREDS=$(echo -n '{"api_key_id":"...","private_key":"-----BEGIN RSA PRIVATE KEY-----\n..."}' | base64)

curl -X POST "https://api.parsecapi.com/api/v1/orders?exchange=kalshi" \
  -H "X-API-Key: pk_live_YOUR_BUILDER_KEY" \
  -H "X-Builder-Customer-Id: CUSTOMER_ID" \
  -H "X-Exchange-Credentials: $CREDS" \
  -H "Content-Type: application/json" \
  -d '{ "market_id": "KXBTC-26FEB18", "outcome": "Yes", "side": "buy", "price": 0.65, "size": 10 }'
```
</RequestExample>
