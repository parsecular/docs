## Overview

As a builder, you route orders for your end-users using **impersonation**.
Every order endpoint works the same as for regular traders — you just add the
`X-Builder-Customer-Id` header.

<Info>
Before routing orders, ensure the end-user has been onboarded to the
target exchange. See [User Management](/builders/user-management).
</Info>

## Placing an order

`POST /api/v1/orders?exchange={exchange}`

### Parameters

<ParamField query="exchange" type="string" required>
  Exchange identifier: `polymarket` or `kalshi`.
</ParamField>

<ParamField header="X-API-Key" type="string" required>
  Your builder API key.
</ParamField>

<ParamField header="X-Builder-Customer-Id" type="string" required>
  The end-user's `customer_id`.
</ParamField>

<ParamField body="market_id" type="string" required>
  Exchange-native market ID.
</ParamField>

<ParamField body="outcome" type="string" required>
  `"Yes"` or `"No"` (binary), or outcome name (categorical).
</ParamField>

<ParamField body="side" type="string" required>
  `"buy"` or `"sell"`.
</ParamField>

<ParamField body="price" type="number" required>
  Limit price (0.01–0.99 for binary).
</ParamField>

<ParamField body="size" type="number" required>
  Number of contracts.
</ParamField>

<ParamField body="params" type="object">
  Optional exchange-specific parameters.

  | Key | Values | Description |
  |-----|--------|-------------|
  | `order_type` | `gtc` (default), `ioc`, `fok` | Time-in-force / execution type |
</ParamField>

<ParamField body="credentials" type="object">
  Optional [per-request credentials](/getting-started/per-request-credentials). Skips stored credentials.
</ParamField>

### Response

<ResponseField name="id" type="string">
  Order identifier.
</ResponseField>

<ResponseField name="market_id" type="string">
  Market the order was placed on.
</ResponseField>

<ResponseField name="outcome" type="string">
  The outcome being traded.
</ResponseField>

<ResponseField name="side" type="string">
  `"buy"` or `"sell"`.
</ResponseField>

<ResponseField name="price" type="number">
  Order price.
</ResponseField>

<ResponseField name="size" type="number">
  Total order size.
</ResponseField>

<ResponseField name="filled" type="number">
  Amount filled so far.
</ResponseField>

<ResponseField name="status" type="string">
  `pending`, `open`, `filled`, `partially_filled`, `cancelled`, or `rejected`.
</ResponseField>

<ResponseField name="created_at" type="string">
  ISO 8601 creation timestamp.
</ResponseField>

<RequestExample>
```bash curl
curl -X POST "https://api.parsecapi.com/api/v1/orders?exchange=polymarket" \
  -H "X-API-Key: pk_live_YOUR_BUILDER_KEY" \
  -H "X-Builder-Customer-Id: CUSTOMER_ID" \
  -H "Content-Type: application/json" \
  -d '{
    "market_id": "0x1234abcd",
    "outcome": "Yes",
    "side": "buy",
    "price": 0.55,
    "size": 100
  }'
```
```typescript TypeScript
const order = await fetch(
  'https://api.parsecapi.com/api/v1/orders?exchange=polymarket',
  {
    method: 'POST',
    headers: {
      'X-API-Key': BUILDER_KEY,
      'X-Builder-Customer-Id': customerId,
      'Content-Type': 'application/json',
    },
    body: JSON.stringify({
      market_id: '0x1234abcd',
      outcome: 'Yes',
      side: 'buy',
      price: 0.55,
      size: 100,
    }),
  },
);
```
```python Python
resp = requests.post(
    "https://api.parsecapi.com/api/v1/orders",
    headers={
        "X-API-Key": BUILDER_KEY,
        "X-Builder-Customer-Id": customer_id,
    },
    params={"exchange": "polymarket"},
    json={
        "market_id": "0x1234abcd",
        "outcome": "Yes",
        "side": "buy",
        "price": 0.55,
        "size": 100,
    },
)
```
</RequestExample>

<ResponseExample>

## Placing an order with fee escrow

To collect fees on an order, include the `fee_auth` object signed by the
end-user's wallet. See [Fee Escrow](/builders/fee-escrow) for details.

<RequestExample>
```bash
curl -X POST "https://api.parsecapi.com/api/v1/orders?exchange=polymarket" \
  -H "X-API-Key: pk_live_YOUR_BUILDER_KEY" \
  -H "X-Builder-Customer-Id: CUSTOMER_ID" \
  -H "Content-Type: application/json" \
  -d '{
    "market_id": "0x1234abcd",
    "outcome": "Yes",
    "side": "buy",
    "price": 0.55,
    "size": 100,
    "fee_auth": {
      "order_id": "0xabc123...",
      "payer": "0xUserWalletAddress",
      "fee_amount": "250000",
      "deadline": 1740500000,
      "signature": "0x..."
    },
    "payer_address": "0xUserWalletAddress",
    "signer_address": "0xUserWalletAddress"
  }'
```
</RequestExample>

## Listing orders

`GET /api/v1/orders?exchange={exchange}`

<RequestExample>
```bash curl
curl "https://api.parsecapi.com/api/v1/orders?exchange=polymarket" \
  -H "X-API-Key: pk_live_YOUR_BUILDER_KEY" \
  -H "X-Builder-Customer-Id: CUSTOMER_ID"
```
```typescript TypeScript
const orders = await fetch(
  'https://api.parsecapi.com/api/v1/orders?exchange=polymarket',
  { headers: builderHeaders(customerId) },
);
```
```python Python
resp = requests.get(
    "https://api.parsecapi.com/api/v1/orders",
    headers=parsec_headers(customer_id),
    params={"exchange": "polymarket"},
)
```
</RequestExample>

## Cancelling an order

`DELETE /api/v1/orders/:order_id?exchange={exchange}`

If fee escrow is active on the order, the escrowed fee is automatically refunded
to the payer's wallet on cancellation.

<RequestExample>
```bash curl
curl -X DELETE "https://api.parsecapi.com/api/v1/orders/ORDER_ID?exchange=polymarket" \
  -H "X-API-Key: pk_live_YOUR_BUILDER_KEY" \
  -H "X-Builder-Customer-Id: CUSTOMER_ID"
```
```typescript TypeScript
const cancelled = await fetch(
  `https://api.parsecapi.com/api/v1/orders/${orderId}?exchange=polymarket`,
  { method: 'DELETE', headers: builderHeaders(customerId) },
);
```
```python Python
resp = requests.delete(
    f"https://api.parsecapi.com/api/v1/orders/{order_id}",
    headers=parsec_headers(customer_id),
    params={"exchange": "polymarket"},
)
```
</RequestExample>

## Other endpoints

All standard Parsec endpoints work with builder impersonation:

| Endpoint | Method | Description |
|----------|--------|-------------|
| `/positions` | GET | List end-user's positions |
| `/balance` | GET | Get end-user's balance |
| `/fills` | GET | Get end-user's trade history |
| `/orderbook` | GET | Get orderbook (no impersonation needed) |
| `/markets` | GET | List markets (no impersonation needed) |
| `/ping` | GET | Check exchange connectivity |

<Note>
Read-only market data endpoints (`/markets`, `/orderbook`, `/trades`, `/price`,
`/events`) do **not** require `X-Builder-Customer-Id` — they work with just
your builder API key.
</Note>

## Order types

| Type | Behavior | Supported |
|------|----------|-----------|
| `gtc` | Good-til-cancelled (default) | All exchanges |
| `ioc` | Immediate-or-cancel | Polymarket, Kalshi |
| `fok` | Fill-or-kill | Polymarket |

Set via `params.order_type` in the create order request body. Unsupported combinations return `501`.
