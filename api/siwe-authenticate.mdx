title: "SIWE Authenticate"
description: "Authenticate with a signed EIP-4361 SIWE message and receive an API key"
---

`POST /api/v1/auth/siwe`

Verifies a signed [EIP-4361 SIWE](https://eips.ethereum.org/EIPS/eip-4361)
message and returns an API key. This is the headless alternative to
browser-based login — designed for scripts, bots, and SDK integrations.

This endpoint is **unauthenticated** — no API key required.

## Parameters

### Body

<ParamField body="message" type="string" required>
  The full EIP-4361 SIWE message text. Must include a valid `Nonce:` from
  `GET /auth/siwe/nonce` and an `Issued At:` timestamp within 5 minutes of
  server time.
</ParamField>

<ParamField body="signature" type="string" required>
  The EIP-191 `personal_sign` signature of the message, hex-encoded with `0x`
  prefix. Produced by `wallet.signMessage(message)` in ethers.js or
  `account.sign_message()` in eth_account.
</ParamField>

## Response

<ResponseField name="api_key" type="string">
  Parsec API key (`pk_live_...`). Use this in the `X-API-Key` header for all
  subsequent requests.
</ResponseField>

<ResponseField name="customer_id" type="string">
  Unique customer identifier (UUID).
</ResponseField>

<ResponseField name="wallet_address" type="string">
  The authenticated Ethereum address, lowercased.
</ResponseField>

<ResponseField name="is_new" type="boolean">
  `true` if this is a first-time registration, `false` if the wallet was
  previously authenticated. The same wallet always returns the same API key.
</ResponseField>

```bash curl
curl -X POST "https://api.parsecapi.com/api/v1/auth/siwe" \
  -H "Content-Type: application/json" \
  -d '{

    "signature": "0x1234...abcd"
  }'
```
```typescript TypeScript
const resp = await fetch('https://api.parsecapi.com/api/v1/auth/siwe', {
  method: 'POST',
  headers: { 'Content-Type': 'application/json' },

});

```

```python Python
resp = requests.post(

```

## SIWE Message Format

The message must follow [EIP-4361](https://eips.ethereum.org/EIPS/eip-4361):

```
{domain} wants you to sign in with your Ethereum account:
{address}

{statement}

URI: {uri}
Version: 1
Chain ID: {chain_id}
Nonce: {nonce}
Issued At: {iso8601_timestamp}
```

| Field | Requirements |
|-------|-------------|
| `address` (line 2) | Valid Ethereum address: 42 chars, `0x`-prefixed hex |
| `Nonce` | Must match a nonce from `GET /auth/siwe/nonce` |
| `Issued At` | RFC 3339 timestamp within 5 minutes of server time |
| `domain`, `URI`, `Version`, `Chain ID`, `statement` | Parsed but not validated server-side |
