---
title: "Get Trades"
description: "Fetch recent public trades for a specific market outcome"
---

`GET /api/v1/trades`

Returns recent public trades ("tape") for a market identified by its `parsec_id`.

## Error Semantics

- `403` means your API key is not authorized for the request.
- `503` means the upstream exchange is temporarily unavailable (retry with backoff).

## Parameters

<ParamField query="parsec_id" type="string" required>
  Market identifier in the format `{exchange}:{market_id}` (e.g., `kalshi:KXPRESPERSON-28-GNEWS`).
</ParamField>

<ParamField query="outcome" type="string" default="yes">
  Outcome selector. For binary markets this is typically `yes` or `no` (case-insensitive). Required for categorical markets.
</ParamField>

<ParamField query="limit" type="number" default="200">
  Max number of trades to return. Range: 1–500.
</ParamField>

<ParamField query="start_ts" type="number">
  Unix seconds start timestamp (inclusive). Historical data is tier-gated: Free: 5 days, Pro: 30 days, Scale: unlimited. Requests beyond your limit return `400`.
</ParamField>

<ParamField query="end_ts" type="number">
  Unix seconds end timestamp (inclusive).
</ParamField>

<ParamField query="cursor" type="string">
  Opaque pagination cursor from a previous response. Pass `next_cursor` from the previous page to fetch the next page of trades.
</ParamField>

## Response

<ResponseField name="(root)" type="object">
  <Expandable title="properties">
    <ResponseField name="parsec_id" type="string">
      The requested market identifier.
    </ResponseField>
    <ResponseField name="exchange" type="string">
      Exchange name.
    </ResponseField>
    <ResponseField name="outcome" type="string">
      The outcome for this tape.
    </ResponseField>
    <ResponseField name="token_id" type="string">
      Exchange-specific token/asset identifier for the requested outcome (when available).
    </ResponseField>
    <ResponseField name="market_id" type="string">
      Exchange-specific market identifier.
    </ResponseField>
    <ResponseField name="trades" type="MarketTrade[]">
      Array of recent trades, newest-first. Prices are normalized to 0.0–1.0.
      <Expandable title="properties">
        <ResponseField name="id" type="string">
          Exchange-specific trade identifier (when available).
        </ResponseField>
        <ResponseField name="price" type="number">
          Trade price, normalized to 0.0–1.0.
        </ResponseField>
        <ResponseField name="size" type="number">
          Trade size in contracts.
        </ResponseField>
        <ResponseField name="side" type="string">
          Trade side: `buy` or `sell` (when available).
        </ResponseField>
        <ResponseField name="aggressor_side" type="string">
          Aggressor side: `buy` or `sell` (when available).
        </ResponseField>
        <ResponseField name="timestamp" type="string">
          Trade timestamp (ISO 8601 / RFC 3339).
        </ResponseField>
        <ResponseField name="source_channel" type="string">
          Data source channel identifier.
        </ResponseField>
        <ResponseField name="outcome" type="string | null">
          Trade outcome side (e.g. `"Yes"`, `"No"`). Present when the exchange provides this.
        </ResponseField>
        <ResponseField name="yes_price" type="number | null">
          YES-side price from exchange data (never derived). May be `null` if the trade was on the NO side.
        </ResponseField>
        <ResponseField name="no_price" type="number | null">
          NO-side price from exchange data (never derived). May be `null` if the trade was on the YES side.
        </ResponseField>
        <ResponseField name="tx_hash" type="string | null">
          Transaction hash (Polymarket only).
        </ResponseField>
        <ResponseField name="taker_address" type="string | null">
          Taker wallet address (Polymarket `proxy_wallet`). `null` for exchanges that don't expose this.
        </ResponseField>
      </Expandable>
    </ResponseField>
    <ResponseField name="has_more" type="boolean">
      `true` if there are more trades beyond this page. Use `next_cursor` to fetch the next page.
    </ResponseField>
    <ResponseField name="next_cursor" type="string | null">
      Opaque cursor for the next page. Pass as `cursor` in the next request. `null` when there are no more pages.
    </ResponseField>
  </Expandable>
</ResponseField>

## Examples

### Recent trades (default)

<RequestExample>
```bash
curl -H "X-API-Key: YOUR_API_KEY" \
  "https://api.parsecapi.com/api/v1/trades?parsec_id=kalshi:KXPRESPERSON-28-GNEWS&outcome=yes&limit=5"
```
</RequestExample>

### Trades in a time window

```bash
# Fetch trades from a time window (24h example)
curl -H "X-API-Key: YOUR_API_KEY" \
  "https://api.parsecapi.com/api/v1/trades?parsec_id=polymarket:572473&outcome=yes&start_ts=1771545600&end_ts=1771632000&limit=100"
  # start_ts = Feb 20 2026 00:00 UTC, end_ts = Feb 21 2026 00:00 UTC
```

### Paginate through trades

```bash
# Fetch the next page using the cursor from the previous response
curl -H "X-API-Key: YOUR_API_KEY" \
  "https://api.parsecapi.com/api/v1/trades?parsec_id=polymarket:572473&outcome=yes&start_ts=1771545600&end_ts=1771632000&limit=100&cursor=eyJ0cyI6MTczOTM1MDY5OH0"
```

For real-time streaming, use the [WebSocket API](/websocket/overview).
