## Base URL

```
https://api.parsecapi.com/api/v1
```

## Authentication

All API requests require an API key passed via the `X-API-Key` header:

```bash
curl -H "X-API-Key: YOUR_API_KEY" https://api.parsecapi.com/api/v1/markets?exchanges=polymarket
```

## Quick Start

Every market in Parsec has a **`parsec_id`** in the format `{exchange}:{market_id}` (e.g., `polymarket:572473`). Use `/markets` to find a market, then pass its `parsec_id` to any data endpoint.

```bash
# 1. Find a market
curl -H "X-API-Key: YOUR_API_KEY" \
  "https://api.parsecapi.com/api/v1/markets?exchanges=polymarket&search=btc&limit=3"
# → Returns markets with parsec_id, e.g. "polymarket:572473"

# 2. Get live orderbook
curl -H "X-API-Key: YOUR_API_KEY" \
  "https://api.parsecapi.com/api/v1/orderbook?parsec_id=polymarket:572473&outcome=yes"

# 3. Get current price (last 7 days of hourly candles)
curl -H "X-API-Key: YOUR_API_KEY" \
  "https://api.parsecapi.com/api/v1/price?parsec_id=polymarket:572473&interval=1h"

# 4. Get historical trades (24h window — timestamps are Unix seconds)
curl -H "X-API-Key: YOUR_API_KEY" \
  "https://api.parsecapi.com/api/v1/trades?parsec_id=polymarket:572473&start_ts=1771545600&end_ts=1771632000"

# 5. Get historical orderbook snapshots (1h window)
curl -H "X-API-Key: YOUR_API_KEY" \
  "https://api.parsecapi.com/api/v1/orderbook?parsec_id=polymarket:572473&start_ts=1771581600&end_ts=1771585200"
```

<Tip>
**Timestamps** are Unix seconds (not milliseconds). Add `start_ts` to any data endpoint to switch from live to historical mode. Use `cursor` to paginate through large result sets.
</Tip>

## Response Format

Single-resource endpoints return the object directly. List endpoints return a wrapper with pagination:

```json
{
  "markets": [{ ... }],
  "pagination": { "count": 100, "total": 12345, "has_more": true, "next_cursor": "100" }
}
```

On error:

```json
{ "error": "Error message here" }
```

## Multi-Exchange Semantics

Some endpoints accept multiple exchanges at once (CSV on the wire):

- `/markets?exchanges=polymarket,kalshi`

## Rate Limiting

All API requests are rate limited per API key using a token bucket algorithm. Limits depend on your tier:

| Tier | Sustained (req/sec) | Burst |
|------|---------------------|-------|
| Free | 2 | 2 (no burst) |
| Pro | 60 | 120 |

When rate limited, you'll receive a `429 Too Many Requests` response with a `Retry-After` header:

```
HTTP/1.1 429 Too Many Requests
Retry-After: 1
Content-Type: application/json

{ "error": "rate limit exceeded" }
```

The SDKs handle 429 responses automatically with 2 retries and exponential backoff.

## Pagination

Endpoints that return lists support cursor-based pagination:

```bash
# First request
GET /markets?exchanges=polymarket&limit=100

# Response includes cursor
{
  "markets": [{ "...": "..." }],
  "pagination": {
    "count": 100,
    "total": 12345,
    "has_more": true,
    "next_cursor": "100"
  }
}

# Next page
GET /markets?exchanges=polymarket&limit=100&cursor=100
```
