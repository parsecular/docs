## Base URL

```
https://api.parsecapi.com/api/v1
```

## Authentication

All API requests require an API key passed via the `X-API-Key` header:

```bash
curl -H "X-API-Key: YOUR_API_KEY" https://api.parsecapi.com/api/v1/markets?exchanges=polymarket
```

## Response Format

2xx endpoints return the resource directly (no `{ success, data }` envelope):

```json
{ ... }
```

On error:

```json
{ "error": "Error message here" }
```

## Available Endpoints

### Exchange Info
| Method | Endpoint | Description |
|--------|----------|-------------|
| `GET` | [`/exchanges`](/api/list-exchanges) | List available exchanges |
| `GET` | [`/ping`](/api/ping) | Check exchange connectivity |

### Markets
| Method | Endpoint | Description |
|--------|----------|-------------|
| `GET` | [`/markets`](/api/list-markets) | List markets from one or more exchanges |

### Orders
| Method | Endpoint | Description |
|--------|----------|-------------|
| `GET` | [`/orders`](/api/list-orders) | List open orders |
| `POST` | [`/orders`](/api/create-order) | Create a new order |
| `GET` | [`/orders/:id`](/api/get-order) | Get order details |
| `DELETE` | [`/orders/:id`](/api/cancel-order) | Cancel an order |

### Portfolio
| Method | Endpoint | Description |
|--------|----------|-------------|
| `GET` | [`/positions`](/api/list-positions) | List positions |
| `GET` | [`/balance`](/api/get-balance) | Get account balance |
| `GET` | [`/user-activity`](/api/user-activity) | Get user activity across exchanges |

## Multi-Exchange Semantics

Some endpoints accept multiple exchanges at once (CSV on the wire):

- `/markets?exchanges=polymarket,kalshi`

Responses include a `meta` section describing which exchanges were queried and which failed:

```json
{
  "markets": [],
  "pagination": { "count": 0, "total": 0, "has_more": false },
  "meta": {
    "exchanges_queried": ["polymarket", "kalshi"],
    "exchanges_succeeded": ["polymarket"],
    "exchanges_failed": { "kalshi": "KALSHI_API_KEY_ID not configured" }
  }
}
```

## Rate Limiting

The API respects rate limits of underlying exchanges. If rate limited, you'll receive:

```json
{ "error": "rate limited" }
```

## Pagination

Endpoints that return lists support cursor-based pagination:

```bash
# First request
GET /markets?exchanges=polymarket&limit=100

# Response includes cursor
{
  "markets": [{ "...": "..." }],
  "pagination": {
    "count": 100,
    "total": 12345,
    "has_more": true,
    "next_cursor": "eyJwb2x5bWFya2V0IjoiMTAwIn0"
  },
  "meta": { "exchanges_queried": ["polymarket"], "exchanges_succeeded": ["polymarket"] }
}

# Next page
GET /markets?exchanges=polymarket&limit=100&cursor=eyJwb2x5bWFya2V0IjoiMTAwIn0
```
