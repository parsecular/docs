---
title: "Update Credentials"
description: "Update exchange credentials for your API key"
---

`PUT /api/v1/credentials`

Updates the stored exchange credentials for the authenticated API key. Returns 204 on success.

All fields are optional; only include the credentials you want to update. Fields set to `null` are ignored.

## Parameters

<ParamField body="evm_private_key" type="string | null">
  EVM private key for Polymarket order signing. Used to derive the wallet address for CLOB operations.
</ParamField>

<ParamField body="poly_funder" type="string | null">
  Polymarket funder address override. If omitted, derived from `evm_private_key`.
</ParamField>

<ParamField body="poly_signature_type" type="string | null">
  Polymarket signature type. Typically `"EOA"` or `"POLY_GNOSIS_SAFE"`.
</ParamField>

<ParamField body="kalshi_api_key_id" type="string | null">
  Kalshi API key ID from the Kalshi dashboard.
</ParamField>

<ParamField body="kalshi_private_key" type="string | null">
  Kalshi RSA private key (PEM format) for request signing.
</ParamField>

## Response

Returns `204 No Content` on success with no body.

<RequestExample>
```bash
curl -X PUT "https://api.parsecapi.com/api/v1/credentials" \
  -H "X-API-Key: YOUR_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "evm_private_key": "0xabc...",
    "poly_signature_type": "EOA"
  }'
```
</RequestExample>

<RequestExample>
```bash Kalshi
curl -X PUT "https://api.parsecapi.com/api/v1/credentials" \
  -H "X-API-Key: YOUR_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "kalshi_api_key_id": "your-key-id",
    "kalshi_private_key": "-----BEGIN RSA PRIVATE KEY-----\n..."
  }'
```
</RequestExample>
