title: "Create Order"
description: "Place a new order on a prediction market"
---

`POST /api/v1/orders`

Places a new order on the specified exchange.

## Parameters

<ParamField query="exchange" type="string" required>
  Exchange identifier: `polymarket` or `kalshi`.
</ParamField>

### Body

<ParamField body="market_id" type="string" required>
  The market to place the order on.
</ParamField>

<ParamField body="outcome" type="string" required>
  The outcome to trade (e.g., `"Yes"` or `"No"`).
</ParamField>

<ParamField body="side" type="string" required>
  Order side: `"buy"` or `"sell"`.
</ParamField>

<ParamField body="price" type="number" required>
  Limit price between 0 and 1 (e.g., `0.65` for 65 cents).
</ParamField>

<ParamField body="size" type="number" required>
  Order size in contracts/shares.
</ParamField>

<ParamField body="params" type="object">
  Optional exchange-specific parameters.

  | Key | Values | Description |
  |-----|--------|-------------|
  | `order_type` | `gtc` (default), `ioc`, `fok` | Time-in-force / execution type. Unsupported combinations return `501` per exchange. |
</ParamField>

<ParamField header="X-Exchange-Credentials" type="string">
  Optional [per-request exchange credentials](/getting-started/per-request-credentials) (base64-encoded JSON). When provided, Parsec
  creates a transient exchange session instead of using stored credentials.

  **Kalshi:** `{ "api_key_id": "...", "private_key": "..." }`

  **Polymarket:** `{ "clob_api_key": "...", "clob_api_secret": "...", "clob_api_passphrase": "..." }`
</ParamField>

### Builder fields (optional)

These fields are only used by [builders](/builders/overview) routing orders on
behalf of end-users via impersonation.

<ParamField header="X-Builder-Customer-Id" type="string">
  The end-user's `customer_id`. Required for builder impersonation.
</ParamField>

<ParamField body="fee_auth" type="object">
  EIP-712 fee authorization signed by the end-user's wallet. Required to collect
  fees via [fee escrow](/builders/fee-escrow).
  <Expandable title="properties">
    <ParamField body="fee_auth.order_id" type="string">
      `0x`-prefixed hex bytes32 order identifier.
    </ParamField>
    <ParamField body="fee_auth.payer" type="string">
      `0x`-prefixed payer wallet address.
    </ParamField>
    <ParamField body="fee_auth.fee_amount" type="string">
      Fee in USDC base units (6 decimals), as a string.
    </ParamField>
    <ParamField body="fee_auth.deadline" type="number">
      Unix timestamp after which the authorization expires.
    </ParamField>
    <ParamField body="fee_auth.signature" type="string">
      `0x`-prefixed hex EIP-712 signature.
    </ParamField>
  </Expandable>
</ParamField>

<ParamField body="payer_address" type="string">
  End-user's wallet address (fee escrow payer). Builder-only.
</ParamField>

<ParamField body="signer_address" type="string">
  End-user's signing wallet address. Builder-only.
</ParamField>

<ParamField body="affiliate" type="string">
  Affiliate address override. Builder-only.
</ParamField>

## Response

<ResponseField name="(root)" type="Order">
  <Expandable title="properties">
    <ResponseField name="id" type="string">
      Order identifier.
    </ResponseField>
    <ResponseField name="market_id" type="string">
      Market the order was placed on.
    </ResponseField>
    <ResponseField name="outcome" type="string">
      The outcome being traded.
    </ResponseField>
    <ResponseField name="side" type="string">
      `"buy"` or `"sell"`.
    </ResponseField>
    <ResponseField name="price" type="number">
      Order price.
    </ResponseField>
    <ResponseField name="size" type="number">
      Total order size.
    </ResponseField>
    <ResponseField name="filled" type="number">
      Amount filled so far.
    </ResponseField>
    <ResponseField name="status" type="string">
      One of: `pending`, `open`, `filled`, `partially_filled`, `cancelled`, `rejected`.
    </ResponseField>
    <ResponseField name="created_at" type="string">
      ISO 8601 creation timestamp.
    </ResponseField>
    <ResponseField name="updated_at" type="string | null">
      ISO 8601 last update timestamp.
    </ResponseField>
  </Expandable>
</ResponseField>

<RequestExample>
```bash
# Limit order (default)
curl -X POST \
  -H "X-API-Key: YOUR_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "market_id": "0x1234abcd",
    "outcome": "Yes",
    "side": "buy",
    "price": 0.65,
    "size": 100
  }' \
  "https://api.parsecapi.com/api/v1/orders?exchange=polymarket"

# IOC order
curl -X POST \
  -H "X-API-Key: YOUR_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "market_id": "0x1234abcd",
    "outcome": "Yes",
    "side": "buy",
    "price": 0.65,
    "size": 100,
    "params": { "order_type": "ioc" }
  }' \
  "https://api.parsecapi.com/api/v1/orders?exchange=polymarket"
```
</RequestExample>
