`GET /api/v1/orderbook`

Returns orderbook data (bids and asks) for a market identified by its `parsec_id`.

Returns the current **live snapshot** of the orderbook from the exchange.

{/* DEPRECATED (2026-02-24): Historical orderbook mode disabled — upstream indexer offline.
    All historical mode code and documentation preserved below for when it recovers.

    Previously supported two modes:
    1. **Live snapshot** (default): Returns the current orderbook from the exchange.
    2. **Historical snapshots** (`start_ts`): Returns a paginated series of L2 orderbook snapshots over a time range.

<Warning>
**Historical orderbook snapshots are temporarily unavailable.** The upstream exchange
orderbook indexer is currently offline. Use [`/price`](/api/get-price) with `start_ts` for
historical price data, or the live orderbook endpoint below. This page will be updated when
the historical data source is restored.
</Warning>

## Error Semantics

- `403` means your API key is not authorized for the request.
- `503` means the upstream exchange is temporarily unavailable (retry with backoff).

## Parameters

<ParamField query="parsec_id" type="string" required>
  Market identifier in the format `{exchange}:{market_id}` (e.g., `polymarket:572473`).
</ParamField>

<ParamField query="outcome" type="string" default="yes">
  Outcome to fetch: `yes` or `no`. Required for categorical markets.
</ParamField>

<ParamField query="limit" type="number" default="50">
  Max price levels per side. Range: 1-100.
</ParamField>

<ParamField query="depth" type="number" default="50">
  Alias for `limit` (for REST/WS symmetry). If both are provided, `limit` takes precedence.
</ParamField>

{/* DEPRECATED (2026-02-24): Historical mode params — upstream indexer offline.

<ParamField query="start_ts" type="number">
  Unix seconds start timestamp (inclusive). When present, switches to **historical mode** and returns orderbook snapshots instead of a live snapshot. Defaults to 1 day ago if only `end_ts` is provided. Note: historical data is tier-gated (Free: 5 days, Pro: 30 days, Scale: unlimited).
</ParamField>

<ParamField query="end_ts" type="number">
  Unix seconds end timestamp (inclusive). Only used in historical mode. Defaults to now.
</ParamField>

<ParamField query="cursor" type="string">
  Opaque pagination cursor from a previous historical response. Pass `next_cursor` from the previous page to fetch the next page of snapshots.
</ParamField>

*/}

## Response

<ResponseField name="(root)" type="object">
  <Expandable title="properties">
    <ResponseField name="parsec_id" type="string">
      The requested market identifier.
    </ResponseField>
    <ResponseField name="exchange" type="string">
      Exchange name.
    </ResponseField>
    <ResponseField name="outcome" type="string">
      The outcome for this orderbook.
    </ResponseField>
    <ResponseField name="token_id" type="string">
      Exchange-specific token/asset identifier.
    </ResponseField>
    <ResponseField name="market_id" type="string">
      Exchange-specific market identifier.
    </ResponseField>
    <ResponseField name="bids" type="array">
      `[price, size]` pairs, sorted by price descending.
    </ResponseField>
    <ResponseField name="asks" type="array">
      `[price, size]` pairs, sorted by price ascending.
    </ResponseField>
    <ResponseField name="timestamp" type="string | null">
      ISO 8601 timestamp of when the orderbook was fetched.
    </ResponseField>
    <ResponseField name="tick_size" type="number | null">
      Minimum price increment for orders on this market. Omitted when not available from the exchange.
    </ResponseField>
    <ResponseField name="min_order_size" type="number | null">
      Minimum order size in contracts. Omitted when not available from the exchange.
    </ResponseField>
  </Expandable>
</ResponseField>
