title: "Field Mappings"
description: "How exchange-specific fields map to unified types"
---

Parsec normalizes data from different prediction market exchanges into a unified format. This document describes how fields from each exchange map to the unified types.

## Market Field Mappings

### Polymarket

| Polymarket Field | Unified Field | Transformation |
|-----------------|---------------|----------------|
| `id` | `id` | direct |
| `question` | `question` | direct |
| `outcomes` | `outcomes` | JSON array or comma-split string → Vec |
| `outcomePrices` | `prices` | JSON string array → HashMap |
| `volumeNum` / `volume` | `volume` | f64 or string parse |
| `liquidityNum` / `liquidity` | `liquidity` | f64 or string parse |
| `minimum_tick_size` | `tick_size` | f64, default 0.01 |
| `description` | `description` | direct |

**API Endpoint:** `GET https://gamma-api.polymarket.com/markets`

### Kalshi

| Kalshi Field | Unified Field | Transformation |
|--------------|---------------|----------------|
| `ticker` | `id` | direct |
| `title` | `question` | direct |
| `yes_bid` / `no_bid` | `prices` | cents → decimal (÷100) |
| `volume` | `volume` | cents → USD |
| `open_interest` | `liquidity` | contracts → USD approximation |
| `subtitle` | `description` | direct |

**API Endpoint:** `GET https://api.elections.kalshi.com/trade-api/v2/markets`

### Limitless

| Limitless Field | Unified Field | Transformation |
|-----------------|---------------|----------------|
| `slug` / `id` | `id` | direct |
| `title` / `question` | `question` | direct |
| `yesPrice` | `prices["Yes"]` | normalize if > 1.0 (÷100) |
| `noPrice` | `prices["No"]` | normalize if > 1.0 (÷100) |
| `volume` | `volume` | direct |
| `liquidity` | `liquidity` | direct |
| `tokens.yes` / `tokens.no` | `metadata.clobTokenIds` | array [yes_id, no_id] |

**API Endpoint:** `GET https://api.limitless.exchange/markets/active`

## Order Field Mappings

### Status Mapping

Order statuses are normalized across exchanges:

| Polymarket | Kalshi | Limitless | Unified |
|------------|--------|-----------|---------|
| `LIVE` | `resting` | `LIVE` | `Open` |
| `MATCHED` | `executed` | `FILLED` | `Filled` |
| `CANCELLED` | `canceled` | `CANCELLED` | `Cancelled` |
| `PARTIALLY_FILLED` | - | `PARTIALLY_FILLED` | `PartiallyFilled` |

### Side Mapping

| Exchange Value | Unified Value |
|----------------|---------------|
| `BUY` / `buy` / `yes` | `Buy` |
| `SELL` / `sell` / `no` | `Sell` |

## Position Field Mappings

### Polymarket

| Polymarket Field | Unified Field | Transformation |
|-----------------|---------------|----------------|
| `conditionId` | `market_id` | direct |
| `outcome` | `outcome` | direct |
| `size` | `size` | string → f64 |
| `avgPrice` | `average_price` | string → f64 |
| `curPrice` | `current_price` | string → f64 |

**API Endpoint:** `GET https://data-api.polymarket.com/positions`

### Limitless

| Limitless Field | Unified Field | Transformation |
|-----------------|---------------|----------------|
| `market_slug` | `market_id` | direct |
| `outcome` | `outcome` | direct |
| `size` | `size` | direct |
| `average_price` | `average_price` | direct |
| `current_price` | `current_price` | direct |

**API Endpoint:** `GET https://api.limitless.exchange/portfolio/{address}/positions`

## Accessing Raw Data

If you need access to exchange-specific fields not included in the unified model, use the `metadata` field on Market objects:

```json
{
  "id": "0x1234",
  "question": "...",
  "metadata": {
    "clobTokenIds": ["token_yes", "token_no"],
    "slug": "bitcoin-100k",
    "tags": [...]
  }
}
```

The `metadata` field contains the complete raw response from the exchange API.
